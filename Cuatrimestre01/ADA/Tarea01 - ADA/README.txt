En cada carpeta se encuentra el reporte para el numero de problema correspondiente

En la carpeta con el nombre 1, se encuentran los archivos pertenecientes a dicho problema
Para el problema 1 se encuentra un archivo con el nombre de problema01.cpp, 
para la compilacion se deberá correr en terminal el siguiente algoritmo

Para Windows
g++ .\problema01.cpp -o prog
\prog.exe .\prueba_false.txt       ó        \prog.exe .\prueba_true.txt

Para macOS y Linux
g++ ./problema01.cpp -o prog
./prog ./prueba_false.txt       ó        ./prog ./prueba_true.txt

En la carpeta con el nombre 2, se encuentran los archivos pertenecientes a dicho problema
Para el problema 2 se encuentra un archivo con el nombre de problema02-unordered_set.cpp, 
para la compilacion se deberá correr en terminal el siguiente algoritmo

Para Windows
g++ .\problema01.cpp -o prog
\prog.exe .\sin_repetidos.txt       ó        \prog.exe .\con_repetidos.txt

Para macOS y Linux
g++ ./problema02-unordered_set.cpp -o prog
./prog ./sin_repetidos.txt       ó        ./prog ./con_repetidos.txt

Dado sea el caso que se requiera hacer nuevas pruebas con una cantidad de datos mayor, 
se debe para cada proeblema ejecutar y correr los archivos de prueba para generar los nuevos txt

Para Windows
g++ .\prueba.cpp -o prog
\prog.exe

Para macOS y Linux
g++ ./prueba.cpp -o prog
./prog
